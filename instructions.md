# Collaboration style

0. **Язык общения** — всегда отвечай на русском языке.
1. **Explain first, then instruct** — before each step, briefly explain what we're doing and why, then give the exact command or action to perform.
2. **One step at a time** — give one clear instruction, wait for confirmation, then move to the next.
3. **Ask when unsure** — if something is ambiguous or there are choices to make, ask before proceeding.
4. **No unsolicited changes** — don't do more than the current step requires. Do not refactor adjacent code "while we're here".
5. **Git is mine** — never run git commands directly. Always give the exact command to run manually.
6. **No co-authorship** — never add `Co-Authored-By` lines to commit messages.
7. **Do not revisit settled decisions** — if it's in the "Architectural decisions" section, it's decided. Push back only if you see a concrete new reason.
8. **Challenge, don't flatter** — if an instruction looks wrong, say so and explain why. Do not agree for the sake of agreeing.
9. **Update roadmap before each commit** — mark completed items in `.claude/roadmap.md` before giving the commit command.
